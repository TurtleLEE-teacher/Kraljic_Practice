# Kraljic Matrix 실습 시뮬레이션 — 공통 프레임워크

## 1. 전체 흐름 개요

실습은 **2층 구조**로 진행된다.

| 층 | 명칭 | 형태 | 내용 |
|---|---|---|---|
| 1층 | 개인 시나리오 플레이 | 개인 | 4개 사분면 시나리오를 순서대로 풀이 |
| 2층 | 돌발 이벤트 대응 | 개인(토론 후 개별 제출) | 공통 이벤트에 대한 4개 사분면 대응 결정 |

**진행 순서:**

```
[1층] 병목 시나리오 → 레버리지 시나리오 → 전략 시나리오 → 일상 시나리오
                              ↓
[2층] 돌발 이벤트 발생 → 팀 토론 → 개별 대응안 제출
                              ↓
               최종 대시보드 (1층 + 2층 합산 결과)
```

---

## 2. 시나리오 구조 템플릿

모든 사분면 시나리오는 동일한 뼈대를 따른다.

### 2.1 기본 구조

| 항목 | 값 |
|---|---|
| 사분면당 Step 수 | **4 Step** |
| Step당 선택지 수 | **3개** (A / B / C) |
| 피드백 방식 | 매 Step 선택 직후 **즉시 피드백** 제공 |
| 피드백 내용 | ① 선택의 결과 서술 ② 트레이드오프 해설 ③ Kraljic 이론 연결 |

### 2.2 시나리오 진행 흐름 (사분면당)

```
[배경 브리핑]  가상 회사·품목·현재 상황 소개
       ↓
[Step 1]  상황 제시 → 선택지 A/B/C → 선택 → 즉시 피드백
       ↓
[Step 2]  이전 선택과 무관하게 새로운 상황 제시 → 선택 → 피드백
       ↓
[Step 3]  상황 제시 → 선택 → 피드백
       ↓
[Step 4]  상황 제시 → 선택 → 피드백
       ↓
[사분면 소결]  해당 사분면의 핵심 교훈 요약
```

> **설계 원칙:** Step 간 분기(branching)를 두지 않는다. 모든 참여자가 동일한 4개 상황을 마주하되, 선택만 다르게 한다. 이렇게 하면 점수 비교가 공정하고, 구현 복잡도도 낮아진다. 피드백 문구는 선택지별로 다르게 제공하여 학습 효과를 확보한다.

### 2.3 피드백 구성 요소

매 Step의 피드백은 다음 3가지 요소를 포함한다.

1. **결과 서술** (2~3문장): "당신의 선택으로 인해 ~한 결과가 발생했습니다."
2. **트레이드오프 해설** (2~3문장): "이 선택은 ~을 얻는 대신 ~을 포기한 것입니다."
3. **이론 연결** (1~2문장): "병목 품목에서는 ~이 핵심이기 때문에, 이 선택은 ~한 의미를 갖습니다."

## 3. 점수 체계

### 3.1 3개 평가 차원

모든 선택지는 아래 **3개 차원**에서 독립적으로 점수를 받는다.

| 차원 | 약칭 | 설명 | 측정하는 역량 |
|---|---|---|---|
| **비용 효율성** | CE (Cost Efficiency) | 구매 비용 절감, 예산 관리, 재고 비용 최적화 | 단기적 재무 성과 관리 능력 |
| **공급 안정성** | SS (Supply Stability) | 공급 리스크 관리, 공급 연속성 확보, 대체 공급원 관리 | 공급망 리스크 관리 능력 |
| **전략적 가치** | SV (Strategic Value) | 장기적 경쟁우위, 공급사 관계 심화, 혁신 협력, 조직 역량 구축 | 장기적 전략 수립 능력 |

### 3.2 Step당 점수 범위

| 항목 | 값 |
|---|---|
| 차원당 점수 범위 | **1 ~ 5** (정수) |
| Step당 최대 점수 | 5 + 5 + 5 = 15 (원점수 합계) |
| 사분면당 최대 원점수 | 4 Step × 15 = 60 |

> 점수 1은 "해당 차원에서 매우 불리한 선택", 5는 "해당 차원에서 최적의 선택"을 의미한다. 모든 차원에서 동시에 5점을 받는 선택지는 존재하지 않는다 — 이것이 트레이드오프의 핵심이다.

### 3.3 사분면별 가중치

각 사분면은 Kraljic 이론에 따라 **중요한 차원이 다르다**. 가중치를 통해 이를 반영한다.

| 사분면 | CE 가중치 | SS 가중치 | SV 가중치 | 핵심 메시지 |
|---|---|---|---|---|
| **병목** (Bottleneck) | 0.20 | 0.50 | 0.30 | 공급 안정성이 가장 중요. 비용이 낮은 품목이지만 공급 단절 시 피해가 크다 |
| **레버리지** (Leverage) | 0.50 | 0.20 | 0.30 | 비용 절감이 핵심. 공급원이 풍부하므로 경쟁을 활용하라 |
| **전략** (Strategic) | 0.20 | 0.30 | 0.50 | 장기적 전략 가치가 핵심. 파트너십과 혁신 협력이 경쟁력을 좌우한다 |
| **일상** (Non-critical) | 0.50 | 0.15 | 0.35 | 비용과 프로세스 효율이 핵심. 관리 에너지를 최소화하면서 낭비를 줄여라 |

### 3.4 사분면별 가중 점수 산출

```
사분면 가중 점수 = Σ(Step별 CE점수) × CE가중치
                + Σ(Step별 SS점수) × SS가중치
                + Σ(Step별 SV점수) × SV가중치
```

**예시 (병목 사분면):**

| Step | CE | SS | SV |
|---|---|---|---|
| 1 | 2 | 4 | 3 |
| 2 | 3 | 4 | 4 |
| 3 | 3 | 4 | 4 |
| 4 | 3 | 4 | 4 |
| **합계** | **11** | **16** | **15** |

```
병목 가중 점수 = 11 × 0.20 + 16 × 0.50 + 15 × 0.30
             = 2.2 + 8.0 + 4.5
             = 14.7
```

### 3.5 1층 총점

```
1층 총점 = 병목 가중 점수 + 레버리지 가중 점수 + 전략 가중 점수 + 일상 가중 점수
```

| 항목 | 점수 범위 |
|---|---|
| 사분면당 가중 점수 범위 | 최소 약 4.0 ~ 최대 20.0 |
| 1층 총점 범위 | 최소 약 16.0 ~ 최대 80.0 |
| 현실적 분포 (예상) | 45 ~ 65 사이에 대부분 분포 |

### 3.6 점수 해석 가이드

| 구간 | 해석 |
|---|---|
| 65 이상 | 우수 — 각 사분면의 핵심을 정확히 이해하고 최적 전략을 선택 |
| 55 ~ 64 | 양호 — 대체로 적절한 판단이나 일부 사분면에서 핵심을 놓침 |
| 45 ~ 54 | 보통 — 사분면별 차이를 인식하지만 일관된 적용에 미숙 |
| 44 이하 | 미흡 — 사분면별 전략 차별화 인식이 부족, 추가 학습 필요 |

## 4. 이벤트(2층) 메커니즘

### 4.1 이벤트 개요

1층 시나리오 완료 후, **전체 참여자에게 동일한 돌발 이벤트**가 주어진다. 이벤트는 공급망에 영향을 미치는 급격한 외부 변화로, 4개 사분면 품목 모두에 대한 대응 결정을 요구한다.

### 4.2 이벤트 구조

```
[이벤트 상황 발표]  전체 공통 — 1개의 돌발 상황 발생
         ↓
[팀 토론]  팀(또는 소그룹)으로 토론 (시간 제한)
         ↓
[개별 제출]  각 사분면별 대응 선택지 A/B/C 중 1개씩 선택 (총 4개 결정)
         ↓
[이벤트 점수 산출]
```

### 4.3 이벤트 점수 산출

이벤트에서는 각 사분면별로 **1개 선택**을 하며, 해당 사분면의 가중치를 동일하게 적용한다.

| 항목 | 값 |
|---|---|
| 사분면당 이벤트 선택지 | 3개 (A/B/C) |
| 사분면당 점수 범위 | 1~5 (차원별) → 가중 적용 후 최대 5.0 |
| 이벤트 총점 범위 | 최대 20.0 (4사분면 × 5.0) |
| 현실적 분포 | 10 ~ 16 |

```
이벤트 총점 = Σ(각 사분면별 이벤트 CE × CE가중치 + SS × SS가중치 + SV × SV가중치)
```

### 4.4 역전 가능성 분석

| 항목 | 값 |
|---|---|
| 1층 총점 최대 | 80.0 |
| 2층 총점 최대 | 20.0 |
| **최종 총점 최대** | **100.0** |
| 2층 비중 | 20% |
| 1층 현실적 편차 | 약 ±10 |
| 2층 현실적 편차 | 약 ±6 |

> **역전 시나리오 예시:**
> - A 참여자: 1층 60점 + 2층 10점 = 70점
> - B 참여자: 1층 55점 + 2층 16점 = 71점
> - B가 1층에서 5점 뒤졌으나, 2층에서 6점 더 받아 **1점 차 역전**
>
> 2층은 "1층에서 약간 뒤진 사람이 만회할 수 있는 기회"를 제공하되, 1층을 완전히 무의미하게 만들지는 않는다. 1층에서 10점 이상 차이가 나면 2층만으로 역전하기 어렵다.

### 4.5 이벤트 제출 형식

참여자는 다음 형태로 제출한다:

```
┌─────────────────────────────────────────┐
│  돌발 이벤트 대응 카드                    │
│                                         │
│  [병목 품목]    ○ A  ○ B  ○ C           │
│  [레버리지 품목] ○ A  ○ B  ○ C           │
│  [전략 품목]    ○ A  ○ B  ○ C           │
│  [일상 품목]    ○ A  ○ B  ○ C           │
│                                         │
│  [제출하기]                              │
└─────────────────────────────────────────┘
```

## 5. 최종 대시보드

### 5.1 표시 항목

대시보드는 실습 종료 후 참여자에게 보여주는 결과 화면이다.

#### A. 종합 결과

| 표시 항목 | 설명 |
|---|---|
| **최종 총점** | 1층 + 2층 합산 (100점 만점 기준) |
| **순위** | 전체 참여자 중 순위 |
| **등급** | 우수/양호/보통/미흡 (3.6 점수 해석 기준) |

#### B. 사분면별 성과

| 표시 항목 | 설명 |
|---|---|
| **사분면별 가중 점수** | 4개 사분면 각각의 가중 점수 (레이더 차트 또는 막대 차트) |
| **최강 사분면** | 가장 높은 점수를 받은 사분면 |
| **최약 사분면** | 가장 낮은 점수를 받은 사분면 |

#### C. 차원별 분석

| 표시 항목 | 설명 |
|---|---|
| **3차원 원점수 합계** | CE / SS / SV 각각의 원점수 합계 (전 사분면 통합) |
| **차원 성향 프로필** | "비용 중시형" / "안정 중시형" / "전략 중시형" / "균형형" 등 라벨 |
| **차원별 레이더 차트** | 3축 레이더 차트로 강약점 시각화 |

#### D. 이벤트(2층) 결과

| 표시 항목 | 설명 |
|---|---|
| **이벤트 총점** | 2층 점수 |
| **이벤트 순위** | 2층만의 순위 |
| **순위 변동** | 1층 순위 → 최종 순위 변동 (▲▼ 표시) |

#### E. 학습 피드백

| 표시 항목 | 설명 |
|---|---|
| **사분면별 핵심 교훈** | 각 사분면에서 학습자가 놓친 포인트 요약 (선택 기반) |
| **추천 학습 포인트** | 가장 약한 사분면에 대한 보충 설명 |

## 6. 설계 원칙 요약

| 원칙 | 설명 |
|---|---|
| **공정성** | 모든 참여자가 동일한 상황·선택지를 마주함 (비분기 구조) |
| **학습성** | 매 선택 후 즉시 피드백으로 "왜 이 선택이 좋은/나쁜지" 학습 |
| **차별화** | 사분면별 가중치로 "같은 행동도 사분면에 따라 다르게 평가됨"을 체감 |
| **역전 가능** | 2층 이벤트로 순위 뒤집기 가능 → 긴장감·참여 동기 유지 |
| **간결성** | 4 Step × 3 선택지로 충분한 깊이 확보, 과도한 복잡성 배제 |
